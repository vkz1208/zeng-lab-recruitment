# 05. 视觉与体验 SPEC

文档状态：已采纳/生效  
当前版本：v0.1-effective  
创建日期：2026-05-26
生效日期：2026-05-26

## 5.1 总体要求

本产品对视觉质量要求非常高。视觉体验必须体现学术可信度、专业感、现代感和可读性。系统应支持不同设计风格模板，供租户选择使用。

## 5.2 官网视觉要求

- 官网应是平台品牌和产品能力的第一展示面。
- 首屏必须清晰表达平台面向高校教授和学术团队。
- 注册入口清晰，但整体不应显得廉价或模板化。
- 案例和模板展示应突出真实网站效果。
- 套餐和服务项目需要清晰、可比较。

## 5.3 租户网站视觉要求

- 租户网站必须适合教授、PI、实验室和课题组展示。
- 视觉风格应支持多样化，例如极简学术、科技感、国际高校风、自然科学风、人文社科风等。
- 页面排版必须保证论文、成员、项目、新闻等高密度内容的可读性。
- 中英文内容均需有良好排版。
- 移动端必须完整可用。
- 不允许出现文本溢出、按钮文字挤压、内容重叠和布局错乱。

## 5.4 后台视觉要求

- 后台应优先强调效率、清晰和稳定。
- 编辑入口应贴近内容本身，支持直观的所见即所得体验。
- 页面管理、风格切换、域名绑定和统计数据应有清晰的信息层级。
- 后台不应使用过度营销化的视觉结构。

## 5.5 风格插件质量要求

每个风格插件都需要满足：

- 有明确适用场景。
- 有完整桌面和移动端适配。
- 有预览图和基础说明。
- 支持默认页面和动态页面。
- 不因内容长度变化导致布局崩坏。
- 切换风格不破坏租户内容数据。
