# 03. 功能模块 SPEC

文档状态：已采纳/生效  
当前版本：v0.1-effective  
创建日期：2026-05-26
生效日期：2026-05-26

## 3.1 平台官网

平台官网应承担注册和转化功能，至少包括：

- 品牌首页。
- 产品价值说明。
- 面向教授、PI、实验室的应用场景。
- 风格模板或案例展示。
- 套餐介绍。
- 注册入口。
- 登录入口。
- 帮助和联系入口。

官网视觉需要体现高可信度、学术气质和高级感，不能只是 demo 网站的简单复用。

## 3.2 超级管理后台

超级管理后台用于平台拥有者管理全部租户和平台能力，至少包括：

- 租户列表、搜索、筛选。
- 租户详情。
- 租户状态管理。
- 域名管理。
- 套餐管理。
- 服务项目和功能开关。
- 风格插件管理。
- 模板资产管理。
- AI provider 配置引用。
- 运营数据概览。
- 异常和失败任务查看。

## 3.3 租户后台

租户后台用于用户管理自己的网站，至少包括：

- 初始化资料上传。
- AI 草稿预览与确认。
- WYSIWYG 内容编辑。
- 页面管理。
- 内容模块管理。
- 图片和媒体管理。
- 论文、新闻、项目、团队成员等结构化内容管理。
- 风格切换。
- 域名绑定。
- 密码管理。
- 访问统计。
- 自动更新审核队列。

## 3.4 租户公开网站

租户公开网站默认支持以下页面：

- 主页。
- 团队。
- 论文。
- 研究方向。
- 新闻。
- 招生 / 加入。
- 联系。

后续应支持按租户需求动态增加页面，例如：

- 项目。
- 课题组动态。
- 课程。
- 资源。
- 成果。
- 媒体报道。
- 数据集。
- 软件下载。
- 合作与资助。

## 3.5 微信小程序

微信小程序用于轻量提醒和审核，目标不是替代完整后台，而是处理高频低成本决策。

小程序至少包括：

- 登录与身份绑定。
- 候选更新提醒。
- 候选内容详情。
- 原始来源链接。
- AI 摘要和建议发布位置。
- 同意发布。
- 拒绝发布。
- 提交修改意见。
- 审核历史。
- 发布结果提醒。
