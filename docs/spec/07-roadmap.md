## 7.1 第一阶段：产品底座

- 固化 SPEC 文档治理。
- 梳理当前 demo 到正式产品的信息架构。
- 完善官网注册入口和租户初始化体验。
- 强化多租户隔离和超级管理后台。
- 建立风格插件的最小模型。

## 7.2 第二阶段：商业化与租户后台

- 增加套餐模型和功能开关。
- 完善租户后台页面管理、风格切换和域名绑定。
- 建立访问统计基础能力。
- 优化 AI 生成质量和资料解析能力。

## 7.3 第三阶段：自动维护与微信小程序

- 建立新闻、论文等候选内容抓取管道。
- 建立审核队列。
- 开发微信小程序提醒和审核能力。
- 支持审核同意后自动发布。

## 7.4 第四阶段：规模化运营

- 完善付费、账单、额度和套餐升级。
- 增加协作者权限。
- 增加审计日志、监控、备份和异常告警。
- 建立模板市场或风格插件管理体系。

## 7.5 假设与默认选择

- SPEC 采用多文档结构，`docs/SPEC.md` 为总入口。
- 本次只生成中文 SPEC 草案，不直接改动功能代码。
- 文档当前已由项目拥有者确认“已采纳/生效”；后续新增或修改内容仍需先进入待审核状态，经确认后生效。
- 现有 demo 页面作为第一套学术网站模板原型，不视为最终视觉标准。
- 付费、微信小程序、自动抓取、访问统计先写入产品和架构 SPEC，具体实现可分阶段落地。
- 后续所有 SPEC 修改都必须保留历史痕迹，不能删除或覆盖原有内容。
